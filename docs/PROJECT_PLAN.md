# Texas Hold'em Poker - 프로젝트 기획서

## 1. 프로젝트 개요

### 1.1 프로젝트명
**Texas Hold'em Poker** — 웹 기반 싱글플레이어 텍사스 홀덤 포커 게임

### 1.2 목적
브라우저에서 바로 플레이할 수 있는 싱글플레이어 텍사스 홀덤 포커 게임을 개발한다.
1명의 인간 플레이어가 5명의 AI 상대와 대결하며, 실제 포커와 동일한 규칙과 몰입감 있는 UI/UX를 제공한다.

### 1.3 대상 사용자
- 포커를 즐기는 캐주얼 게이머
- 텍사스 홀덤 규칙을 배우고 싶은 초보자
- 짧은 시간에 가볍게 플레이하고 싶은 사용자

### 1.4 핵심 가치
- **접근성**: 설치 없이 웹 브라우저에서 즉시 플레이
- **몰입감**: 애니메이션, 사운드, 개성 있는 AI로 실감나는 경험
- **학습 친화적**: 핸드 랭킹 표시, AI의 다양한 전략을 통해 자연스럽게 학습

## 2. 기능 명세

### 2.1 게임 규칙
| 항목 | 설명 |
|------|------|
| 게임 종류 | 노리밋 텍사스 홀덤 |
| 참가 인원 | 6인 (인간 1명 + AI 5명) |
| 시작 칩 | 10,000 |
| 블라인드 | SB 50 / BB 100 (10핸드마다 1.5배 증가) |
| 승리 조건 | 다른 모든 플레이어를 탈락시키면 승리 |

### 2.2 게임 플로우
```
[게임 시작] → [딜러 버튼 배정]
    ↓
[스몰/빅 블라인드 강제 베팅]
    ↓
[홀 카드 2장 딜] → [프리플롭 베팅]
    ↓
[플롭 3장 공개] → [플롭 베팅]
    ↓
[턴 1장 공개] → [턴 베팅]
    ↓
[리버 1장 공개] → [리버 베팅]
    ↓
[쇼다운] → [팟 분배] → [다음 핸드]
```

### 2.3 베팅 액션
| 액션 | 설명 | 가능 조건 |
|------|------|----------|
| Fold | 핸드를 포기하고 이번 판에서 빠짐 | 항상 |
| Check | 베팅 없이 턴을 넘김 | 현재 베팅 없을 때 |
| Call | 상대의 베팅과 동일한 금액을 냄 | 상대가 베팅했을 때 |
| Raise | 상대보다 더 높은 금액을 베팅 | 칩이 충분할 때 |
| All-In | 보유 칩 전부를 베팅 | 항상 |

### 2.4 핸드 랭킹 (높은 순)
| 순위 | 핸드 | 설명 |
|------|------|------|
| 1 | Royal Flush | A-K-Q-J-10 같은 무늬 |
| 2 | Straight Flush | 연속 5장 같은 무늬 |
| 3 | Four of a Kind | 같은 숫자 4장 |
| 4 | Full House | 같은 숫자 3장 + 2장 |
| 5 | Flush | 같은 무늬 5장 |
| 6 | Straight | 연속 5장 |
| 7 | Three of a Kind | 같은 숫자 3장 |
| 8 | Two Pair | 페어 2개 |
| 9 | One Pair | 같은 숫자 2장 |
| 10 | High Card | 위에 해당 없음, 가장 높은 카드 |

### 2.5 사이드팟
올인 플레이어가 있을 때 사이드팟이 생성된다.
- 올인 금액까지가 메인팟, 초과분은 사이드팟
- 각 팟은 해당 팟에 참여 자격이 있는 플레이어들끼리만 경쟁
- 여러 명이 다른 금액으로 올인하면 다중 사이드팟 발생

## 3. AI 시스템

### 3.1 AI 프로필

| 이름 | 성격 | 특징 |
|------|------|------|
| **Alex** | Tight-Aggressive | 좋은 핸드만 플레이하지만 공격적으로 베팅. 중상급 |
| **Bella** | Loose-Aggressive | 많은 핸드를 플레이하며 블러핑을 자주 시도. 예측 어려움 |
| **Charlie** | Tight-Passive | 보수적으로 플레이. 강한 핸드가 아니면 콜만 함 |
| **Diana** | Loose-Passive | 많은 핸드를 콜하지만 레이즈는 잘 안 함. 초보 스타일 |
| **Eddie** | Balanced | 균형 잡힌 전략. 상황에 맞게 유동적으로 플레이. 가장 강함 |

### 3.2 AI 의사결정 구조
1. **프리플롭**: Chen Formula로 시작 핸드 강도를 계산하여 플레이 여부 결정
2. **포스트플롭**: 현재 핸드 강도 + 드로우 가능성 + 팟 오즈를 종합 평가
3. **성격 반영**: 각 AI의 tightness, aggression, bluffFrequency 파라미터로 결정 가중치 조절
4. **스킬 차이**: 높은 스킬의 AI는 정확한 판단, 낮은 스킬은 랜덤성 증가

## 8. 주요 알고리즘

### 8.1 핸드 평가 알고리즘
- 7장에서 C(7,5) = 21개의 5장 조합 생성
- 각 조합을 Royal Flush부터 High Card까지 순서대로 판별
- 단일 숫자(rankValue)로 인코딩하여 O(1) 비교
- 동일 카테고리 내에서는 키커로 승부

### 8.2 Chen Formula (프리플롭 핸드 강도)
1. 높은 카드의 기본 점수: A=10, K=8, Q=7, J=6, 10~2=rank/2
2. 페어: 기본점수 × 2 (최소 5)
3. 수티드(같은 무늬): +2
4. 갭 패널티: 갭1=-1, 갭2=-2, 갭3=-4, 갭4+=-5
5. Q 미만이면서 갭 0~1: +1 (스트레이트 보너스)

### 8.3 사이드팟 알고리즘
1. 각 플레이어의 총 베팅액을 기록
2. 올인 금액을 오름차순 정렬
3. 각 올인 레벨에서: (현재레벨 - 이전레벨) × 참여인원 = 해당 팟 금액
4. 마지막 레벨 이후 잔액이 있으면 최종 팟에 추가
