# PRD: Texas Hold'em Poker

## 1. 제품 개요

| 항목 | 내용 |
|------|------|
| **제품명** | Texas Hold'em Poker |
| **유형** | 웹 브라우저 싱글플레이어 포커 게임 |
| **대상** | 캐주얼 게이머, 포커 입문자 |
| **핵심 가치** | 설치 없이 브라우저에서 즉시 플레이, 개성 있는 AI 5명과 대전 |

## 2. 문제 정의

포커를 배우거나 연습하고 싶지만, 실제 돈을 걸기는 부담스럽고 멀티플레이어 매칭은 번거로운 사용자를 위한 가벼운 싱글플레이어 포커 게임이 필요하다. 기존 웹 포커 게임은 무거운 프레임워크와 외부 의존성으로 로딩이 느리거나, AI가 단순해서 재미가 없다.

## 2.1 기존 솔루션과의 차별화

| 기존 솔루션 | 사용자 수 | 문제점 | 우리의 차별점 |
|------------|----------|--------|-------------|
| PokerStars | 월 1억+ | 실제 돈 필요, 매칭 대기 30초+, 500MB+ 클라이언트 | 무료, 즉시 플레이, 설치 없음 |
| Zynga Poker | 월 3,800만 | 앱 설치 필요, 전면 광고, 인앱 결제 유도 | 브라우저만으로 플레이, 광고 없음 |
| Governor of Poker 3 | 월 500만 | AI가 랜덤(GTO 미적용), WebGL 기반 무거운 로딩(5초+) | Chen Formula + 팟 오즈 AI, 0.5초 로딩 |
| free-poker-games.net | 월 10만 | AI가 단순 확률 기반, Flash→JS 전환 불안정, 모바일 미대응 | 5개 성격 프로필, 네이티브 반응형 |
| 프레임워크 기반 게임 | - | React/Vue 등 200KB+ 번들, 느린 TTI(3초+) | gzip 18KB, TTI 0.5초 이내 |

### 시장 맥락

글로벌 온라인 포커 시장은 2024년 기준 약 90억 달러 규모이며, 모바일/웹 기반 캐주얼 포커가 빠르게 성장 중. 특히 **연습/학습 목적의 무료 포커**는 입문자 유입 경로로 수요가 높으나, 기존 솔루션은 (1) 과도한 수익화(광고/인앱결제) 또는 (2) 낮은 AI 품질로 사용자 경험이 저하됨.

### 타겟 사용자 니즈 검증

- **포커 입문자**: 실제 돈 부담 없이 규칙을 배우고 싶다 → 핸드 랭킹 도움말 + 단계적 AI 난이도
- **캐주얼 게이머**: 5~15분 짧은 세션으로 가볍게 즐기고 싶다 → 빠른 로딩 + 블라인드 자동 증가로 게임 시간 제한
- **개발자/학습자**: 포커 AI 알고리즘을 이해하고 싶다 → Chen Formula, 팟 오즈 등 학술적 알고리즘 오픈소스 공개

**핵심 차별 요소**:
1. **초경량 번들** — 외부 런타임 의존성 없이 gzip 18KB. 어떤 기기에서든 즉시 로딩
2. **학술적 AI 알고리즘** — Chen Formula(프리플롭) + 핸드 강도 정규화 + 팟 오즈(포스트플롭)를 결합한 5개 성격 프로필
3. **완전한 코드 합성** — 카드(CSS), 사운드(Web Audio), 애니메이션(Web Animations API) 모두 외부 에셋 없이 코드로 생성

## 3. 목표

- **P0**: 1인 vs AI 5명 노리밋 텍사스 홀덤을 완전하게 플레이할 수 있다
- **P0**: 모든 포커 규칙이 정확히 구현됨 (핸드 랭킹, 사이드팟, 블라인드)
- **P1**: 5명의 AI가 각기 다른 전략으로 플레이하여 현실감 있는 대전 경험
- **P1**: 카드 딜링, 칩 이동 등 시각적 애니메이션으로 몰입감 제공
- **P2**: 사운드 효과, 승리 연출, 반응형 디자인

## 4. 사용자 시나리오

### 시나리오 1: 첫 게임 시작
1. 사용자가 브라우저에서 접속하면 시작 화면이 표시된다
2. "게임 시작" 버튼을 클릭하면 10,000칩을 가지고 게임이 시작된다
3. 6인 포커 테이블이 렌더링되고, 딜러 버튼이 배정된다

### 시나리오 2: 베팅 라운드
1. 자신의 차례가 되면 베팅 컨트롤(Fold/Check/Call/Raise)이 활성화된다
2. Raise 시 슬라이더 또는 프리셋 버튼(1/2 Pot, Pot, All-In)으로 금액을 선택한다
3. 액션 후 AI들이 순서대로 의사결정을 하며, 각 액션이 화면에 표시된다

### 시나리오 3: 쇼다운 & 승리
1. 리버까지 진행 후 남은 플레이어의 카드가 공개된다
2. 최고 핸드의 플레이어에게 팟이 분배된다
3. 핸드 결과(핸드 이름, 획득 칩)가 표시된 후 다음 핸드로 넘어간다

### 시나리오 4: 게임 종료
1. 다른 모든 플레이어를 탈락시키면 최종 승리 화면이 표시된다
2. 자신의 칩이 0이 되면 패배 화면이 표시된다

## 5. 기능 요구사항

### 5.1 게임 규칙 (P0)
| ID | 기능 | 설명 |
|----|------|------|
| GR-01 | 게임 종류 | 노리밋 텍사스 홀덤 |
| GR-02 | 참가 인원 | 6인 (인간 1명 + AI 5명) |
| GR-03 | 시작 칩 | 10,000 |
| GR-04 | 블라인드 | SB 50 / BB 100, 10핸드마다 1.5배 증가 |
| GR-05 | 승리 조건 | 다른 모든 플레이어를 탈락시키면 승리 |
| GR-06 | 핸드 랭킹 | Royal Flush ~ High Card (10단계) |
| GR-07 | 사이드팟 | 올인 시 사이드팟 자동 생성 및 분배 |

### 5.2 베팅 시스템 (P0)
| ID | 기능 | 설명 |
|----|------|------|
| BT-01 | Fold | 핸드 포기 |

| BT-05 | All-In | 보유 칩 전액 베팅 |
| BT-06 | 프리셋 | 1/2 Pot, 3/4 Pot, Pot, All-In 버튼 |

### 5.3 AI 시스템 (P1)
| ID | 기능 | 설명 |

| AI-02 | 프리플롭 | Chen Formula 기반 핸드 강도 계산 |
| AI-03 | 포스트플롭 | 핸드 강도 + 팟 오즈 기반 의사결정 |

### 9.2 검증 프로세스

**Phase 1: 자체 검증** (완료)

- 8개 스프린트별 기능 검증 (수동 테스트)
- 82개 단위 테스트 + 커버리지 리포트
- GitHub Actions CI/CD 자동 검증

**Phase 2: 피어 리뷰** (계획)

- 대상: 3~5명의 테스터
- 테스트 태스크:
  1. 게임 시작 → 최소 5핸드 플레이 → 결과 확인
  2. 설정 메뉴에서 사운드/애니메이션 조절
  3. 모바일 브라우저에서 플레이
  4. 핸드 랭킹 도움말 활용
- 수집 데이터: 완료 여부, 플레이 핸드 수, 체감 난이도, UI 직관성 (5점 척도)

**Phase 3: 피드백 반영** (계획)

- 피어 리뷰 결과 분석
- AI 밸런스 파라미터 조정
- UX 개선점 반영
